/// details | 什么是异步函数？

**异步编程** 是一种让程序在等待某些操作（比如网络请求）完成时，可以继续做其他事情的编程方式。

想象你在餐厅点餐：

* **同步方式**：你点完餐后，就站在柜台前一直等，直到餐点做好。在这期间你什么都不能做。
* **异步方式**：你点完餐后拿到一个号码牌，可以去找座位、玩手机，等餐点好了再去取。

异步编程让你的程序更高效，尤其是在处理网络请求时。SeerAPI 使用异步 API，这意味着它可以同时处理多个请求，性能更好。

**`async` 和 `await` 的关系** 🔗

在 Python 中，`async` 和 `await` 是一对好搭档，它们必须配合使用：

* **`async`** - 用来 **定义** 异步函数，声明"这个函数里面会有异步操作"
  ```python
  async def main():  # 用 async 定义异步函数
      # 这里可以使用 await
  ```

* **`await`** - 用来 **等待** 异步操作完成，只能在异步函数（`async def`）内部使用
  ```python
  skill = await client.get('skill', 38088)  # 用 await 等待异步操作
  ```

**它们的关系就像钥匙和锁：** 🔑🔒

* `async` 打开了一扇门（创建异步函数），让你可以在里面使用 `await`
* `await` 是你在这扇门里面才能使用的工具，它会"暂停"在这里，等待结果返回
* 如果没有 `async` 定义函数，你就不能使用 `await`
* 如果有 `async` 但不使用 `await`，函数就失去了异步的意义

**简单记忆法：**
```python
async def 函数名():     # async 说："我是异步函数"
    结果 = await 操作()  # await 说："我要等这个操作完成"
```

继续用餐厅比喻：

* `async def` 就像进入"可以点外卖的餐厅"🍤
* `await` 就像"等待外卖送到"🎁
* 在普通餐厅（普通函数）里，你不能点外卖（不能用 `await`）❌

> 💡 **提示**：如果你对异步编程不熟悉，别担心！按照示例代码写就行了，你会慢慢理解的。记住这个模式：`async def` 定义函数，`await` 调用异步操作。

### 第 3 步：创建 SeerAPI 客户端

现在是关键部分，我们创建一个 SeerAPI 客户端实例：

{* ./src/tutorial/1.开始使用/001_py310.py ln[4:5] hl[5]*}

这里我们使用了 `async with` 语句。`SeerAPI()` 创建了一个客户端实例，它是一个 **异步上下文管理器** 。

/// details | 什么是上下文管理器？

**上下文管理器** 是 Python 的一个强大特性，用于管理资源的生命周期。

还是用餐厅的比喻：

* **不使用上下文管理器**：你需要手动"开门进餐厅"、"吃饭"、"关门离开餐厅"
* **使用上下文管理器**：你只需要说"我要在餐厅里吃饭"，进门和离开都自动帮你处理好

在 Python 中，上下文管理器通过 `with` 语句使用，例如`open`函数实际上返回的就是一个上下文管理器：

```python
# 不使用上下文管理器（需要手动关闭）
file = open('data.txt')
content = file.read()
file.close()  # 容易忘记！如果出错，可能不会执行

# 使用上下文管理器（自动关闭）
with open('data.txt') as file:
    content = file.read()
# 文件会自动关闭，即使发生错误也会正确清理
```

对于异步上下文管理器，我们使用 `async with`：

```python
async with SeerAPI() as client:
    # 在这个代码块中使用 client
    skill = await client.get('skill', 38088)
# 当代码块结束时，client 会自动关闭连接并清理资源
```

/// details | 什么是列表推导式？

**列表推导式** 是 Python 的一个强大特性，可以用一行代码创建列表。

上面的代码：
```python
[effect.info for effect in skill.skill_effect]
```

等价于这样的传统写法：
```python
effect_list = []
for effect in skill.skill_effect:
    effect_list.append(effect.info)
# 然后使用 effect_list
```

可以看到，列表推导式让代码更简洁！🤓它的基本语法是：
```python
[表达式 for 变量 in 可迭代对象]
```

在我们的例子中：

* **表达式**：`effect.info` - 我们想要的值
* **变量**：`effect` - 循环中的当前项
* **可迭代对象**：`skill.skill_effect` - 我们要遍历的列表

所以整行代码的意思是："从 `skill.skill_effect` 列表中取出每个效果的 `info` 属性，组成一个新列表，然后用换行符连接起来。"😎

> 💡 **提示**：如果你对列表推导式不熟悉，使用传统的 for 循环也完全没问题！它们的效果是一样的。

## 快速上手模板 📋

如果你想快速尝试SeerAPI，这是一个通用的模板：

```python
from seerapi import SeerAPI
import asyncio


async def main():
    async with SeerAPI() as client:
        # 在这里写你的代码
        # 例如：
        # pet = await client.get('pet', 2)  # 获取精灵
        # item = await client.get('item', 1000001)  # 获取道具
        pass


if __name__ == '__main__':
    asyncio.run(main())
```

只需要替换注释部分的代码，就能开始使用 SeerAPI 了！
